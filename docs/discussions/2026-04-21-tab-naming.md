# 2026-04-21 「进行中」tab 文案调整提案（A21-14）

**Owner（联签）：** Design + PM
**Decision Due：** 2026-04-24（本周内）
**关联：** A21 晨会 Design 反馈，P0-07 修复后产物
**优先级：** P2（小文案，但影响用户认知）

---

## 现状

- 入口：小程序 `wechat/pages/orders/list`，订单列表顶部 tab 切换组件。
- 当前文案：`待支付 / 进行中 / 已完成 / 已取消`。
- 「进行中」tab 在 P0-07 修复（订单状态分组聚合）后，**同时包含**两类后端状态：
  - `accepted`：陪诊师已接单但尚未上门
  - `in_progress`：陪诊师已开始服务

## 问题

「进行中」在中文语义上更偏向「服务正在发生」，但目前还把「已接单未上门」也归入此 tab。
- 用户视角：刚下完单看到「进行中」，会以为陪诊师已经在路上甚至已到医院，但实际可能只是接单确认。
- 客服反馈风险：上门时间相关的咨询会因预期错位增加。
- Design 在 A21 晨会明确提出此体验风险。

## 方案

### 方案 1：tab 名直接改为「服务中」（推荐）
- 一字之差，覆盖「已接单 → 上门 → 服务中」整个履约阶段。
- 实施成本：~0（文案 key 替换）。
- 不需要后端改动，不需要状态拆分。

### 方案 2：tab 名维持「进行中」，下方加状态标签
- 在订单卡片上加 `已接单` / `服务中` 子状态 badge。
- 信息更精确，但实施成本更高（卡片组件改造 + 视觉规范），且 tab 文案问题没解决。
- 适合 v2 详细化场景。

### 推荐：方案 1
一字之差、0 实施成本、语义更准。方案 2 可作为 v2 增强项独立排期，与 tab 改名并不冲突。

## 影响面

- `wechat/` 订单列表 tab 文案 key（i18n / 常量）。
- **需 PM 同步对齐**：iOS App 订单列表（若已存在「进行中」tab）、运营后台 dashboard 订单筛选项的对应文案。
- 如果其它端短期不跟改，至少先在小程序落地，避免文案口径在内部文档/客服话术上分裂。
