## 重新评估机制

1. **⏸️ 观察项**：触发条件达成后由 Tech Lead 邮件/系统通知用户重新评估，确认后自动进入排期
2. **❌ 暂不做项**：用户可随时申请重新评估，Tech Lead 在 48h 内给出工时评估和排期建议
3. **触发条件达成自动激活流程**：
   - Tech Lead 每周五 check 触发条件状态
   - 条件达成 → 创建功能 brief → 用户确认 → 写 D-XXX 决策记录 → 进入 Sprint Backlog
4. **详细功能 brief 和路线图**：见 docs/PRODUCT_BACKLOG.md
