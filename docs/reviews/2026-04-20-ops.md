## 1. 总体评价（部署成熟度 4/10）

项目在**架构设计层面**做了大量前瞻规划（observability 规范、deployment.md 详尽的 Azure 部署方案、蓝绿发布流程、回滚矩阵），但**实际落地程度极低**——几乎所有运维基础设施停留在"文档占位 + 骨架代码"阶段。五项外部凭证/资源 Blocker（B-01 ~ B-05）全部 Pending，意味着当前无法部署任何非本地环境。

**亮点**：
- config.py 生产模式校验（D-006）拒绝弱密钥/debug 启动，属于安全左移的好实践
- readiness/liveness 双探针分离（D-021），readiness 检查 DB + Redis
- CI 三层防护（test.yml + ci-smoke.yml + pre-commit alembic check）
- deploy.yml 骨架完整，staging → production 带审批门禁
- PG advisory lock 替代 Redis SET NX（D-018 Update），消除分布式锁安全隐患

**短板**：无真实环境部署经验、无监控告警实例、无安全扫描、无备份演练、无压测基线。

## 2. 部署与配置现状

### 2.1 Dockerfile

- 基于 `python:3.11-slim`，15 行极简构建。
- **问题**：
  - 无多阶段构建，镜像含 pip + 编译工具链，体积偏大
  - 无 `USER` 指令，容器以 root 运行（**P1 安全风险**）
  - 无 `HEALTHCHECK` 指令（依赖外部探针，可接受但不理想）
  - 未固定 pip 包版本哈希（`--require-hashes`），供应链攻击面

### 2.2 docker-compose.yaml

- 仅本地开发用途，PostgreSQL 15 + Redis 7，healthcheck 配置合理。
- `POSTGRES_PASSWORD: postgres` 硬编码（dev 可接受）。
- **问题**：`volumes: ./app:/app/app` 挂载源码，prod 不应使用此 compose 文件——当前无 `docker-compose.prod.yml` 区分。

### 2.3 配置管理

- `config.py` 使用 pydantic-settings，支持 `.env` 文件加载。
- `admin_api_token` 默认值 `dev-admin-token` 在生产校验器中**未检查**（仅检查 JWT/debug/payment/sms），存在后门风险。
- `.env.example` 缺少 `ADMIN_API_TOKEN`、`SCHEDULER_ENABLED`、`WS_PUBSUB_ENABLED`、`WS_MAX_CONNECTIONS_PER_USER` 等新增配置项，与 config.py 已脱节。

### 2.4 Alembic 迁移

- 17 个 revision，最新为 `d9e0f1a2b3c4_add_admin_audit_logs`。
- ci-smoke.yml 含真 PG 冒烟测试 + `alembic check` 漂移检测。
- **问题**：TD-OPS-02 指出 readiness 不检查迁移漂移——部署新代码忘跑迁移时 pod 仍 200 ready。

### P1 — 严重隐患

| # | 问题 | 影响 | 建议 |
|---|------|------|------|
| P1-1 | Dockerfile 以 root 运行 | 容器逃逸攻击面 | 添加 `RUN adduser --system app` + `USER app` |

| P1-5 | 无密钥轮换机制 | JWT 密钥泄露后无法快速止损 | 文档化轮换 SOP + 双密钥过渡方案 |

## 5. 容灾与降级缺失

### 5.1 单点故障

| 组件 | 现状 | 风险 |
|------|------|------|
| API 实例 | docker-compose 单实例 | 进程崩溃 = 全站不可用 |
| PostgreSQL | docker-compose 单节点 | 数据丢失风险 |
| Redis | 单节点 | OTP/限流/WS Pub/Sub 全部中断 |

deployment.md 规划了 ACA 多副本 + PG 区域冗余 + Redis Standard，但**均未创建**。

### 5.2 外部依赖降级

| 依赖 | 失败时行为 |
|------|-----------|
| 微信支付 | mock provider 永远成功；wechat provider 未实现 `query`，无法主动查单 |
| 阿里云 SMS | `AliyunSMSProvider` 直接 `raise NotImplementedError`（D-024 设计如此）——**生产配为 aliyun 时启动即炸** |
| Redis 不可用 | WS Pub/Sub 降级为单机；APScheduler 锁退化；OTP 限流退化为进程内 |

**缺失**：无熔断器包裹外呼链路（outbound 装饰器存在但未集成到 SMS/Payment 调用路径），无降级 fallback UI。

### 5.3 备份恢复

- 从未执行过恢复演练
